## Durable Rules Adopted

### 1. Quantify CEX Outflow Fan-Out

CEX outflow into several recipients is a discovery event. Common-control confidence increases when the transfers are near-equal, share one source, occur inside a short window, land in fresh-wallet candidates, share gas funding, and later move in sync.

Required fields:

- `source_exchange`
- `exchange_label_quality`
- `outflow_usd`
- `outflow_pct_mc`
- `recipient_count`
- `fresh_wallet_count`
- `equal_tranche_cv`
- `common_gas_source_ratio`
- `window`
- `public_signal_lag`
- `next_hop`
- `cex_redeposit`
- `quote_recovery`

Direction remains `unknown` at fan-out time. Ordinary retail withdrawals, exchange internal routing, custody movement, and later distribution can create the same first hop.

Representative sources:

- `https://x.com/Miles082510/status/2074772990227865911`: PARTI, 20+ new-wallet claim, about $27K each.
- `https://x.com/Miles082510/status/2076605860793770334`: B2, 28 recipients and 19 claimed fresh wallets.
- `https://x.com/Miles082510/status/2066393953075880130`: BANK, 21-wallet distribution claim.
- `https://x.com/Miles082510/status/2068929872236515659`: XPIN, ten-wallet withdrawal claim.
- `https://x.com/Miles082510/status/2067242926292504858`: BLUAI, one large CEX outflow after a test transfer.

Action mapping:

- Fan-out alone: `Observe`.
- Verified synchronized accumulation plus market confirmation and clean next hop: `Follow only after confirmation`.
- CEX redeposit, DEX distribution, quote recovery, or sibling-cluster conflict: `Reduce/Avoid` according to held exposure and exit liquidity.

### 2. Add An Operator-Level Cluster Conflict Gate

Track clusters as related components of an operator only after entity linkage is verified. A single cluster can support price, accumulate, wash volume, route withdrawals, distribute, or create a misleading local signal.

Required fields:

- `cluster_role`: `accumulating / supporting / washing / distributing / routing / unknown`
- `cluster_balance_delta`
- `operator_phase`: `accumulating / supporting / washing / distributing / routing / mixed / unknown`
- `phase_changed_at`
- `operator_netflow`
- `conflicting_cluster_count`
- `cluster_conflict`

Gate:

- Active verified distribution in any sibling cluster blocks `Small test` and `Follow only after confirmation` language generated by a separate accumulation cluster.
- A mixed operator state stays `Observe`.
- Held exposure enters `Reduce watch` when distribution is verified and exit liquidity remains available.
- Missing entity linkage keeps the operator state `unknown`.

Representative sources:

- `https://x.com/Miles082510/status/2076490669611495523`: PARTI distribution followed by renewed CEX outflow.
- `https://x.com/Miles082510/status/2076605860793770334`: same-day PARTI warning after the renewed-withdrawal thesis.

### 3. Record Public Signal Lag

Every case needs four timestamps when available:

1. original event time;
2. first public post time;
3. first actionable confirmation time;
4. outcome-review time.

The lag changes the attainable trade. BANK's public signal arrived about ten days after the claimed transfer. XPIN's first post explicitly described a move already visible, while the later withdrawal update was the cleaner public checkpoint.

Action mapping:

- Retrospective discovery: case evidence only.
- Public signal after a large price move: `Observe/Avoid` until a reset and new demand appear.
- Fresh signal with frozen timestamp: forward-case monitoring.

### 4. Evaluate Instrument Suitability For Long Incubation

The account describes waits ranging from weeks to months, thin pool capacity, violent wicks, and a preference for spot in high-control setups. Treat those durations as sample observations.

Ask:

- Can the intended size exit within measured depth and slippage?
- What funding carry accumulates through the configured time stop?
- Can a wick hit liquidation or an unexecutable stop?
- Does the venue support a clean spot exit?
- What evidence invalidates the wait before the time stop?

Keep take-profit and time-stop values configurable per position. The author's 10-20% exit heuristic does not become a global setting.
